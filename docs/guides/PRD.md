# AWS 데이터 전송비용 이상 감지 및 자동조치 시스템 MVP PRD

## 핵심 정보

**목적**: AWS 데이터 전송비용이 평소보다 급증했을 때 원인을 AI가 분석하고, 관리자 승인을 거쳐 자동 조치까지 실행하는 이상 감지 및 대응 시스템
**사용자**: AWS 인프라를 운영하는 DevOps 엔지니어 및 시스템 관리자

---

## 사용자 여정

```
1. 이상 감지 목록 페이지 (랜딩)
   ↓ 페이지 진입 시 Notion API에서 Fake 이상 감지 데이터 로드
   ↓ 비용 급증 항목 카드 리스트 표시

2. 이상 항목 선택 (카드 클릭)
   ↓

3. AI 분석 상세 페이지
   ↓ 선택한 항목의 비용 급증 구간, 원인, AI 분석 결과 표시
   ↓ AI 제안 조치 목록 표시
   ↓ "관리자 승인 요청" 버튼 클릭
   ↓

4. 관리자 승인 대기 상태 표시 (AI 분석 상세 페이지 내 상태 변경)
   ↓ 관리자 승인 페이지로 이동 (헤더 메뉴 또는 알림)
   ↓

5. 관리자 승인 페이지
   ↓ 승인 대기 중인 조치 목록 표시
   ↓ [승인] 버튼 클릭

   [승인] → 자동 조치 실행 (Fake 처리) → 장애 리포트 자동 생성 → 장애 리포트 페이지로 이동
   [거부] → 상태를 "거부됨"으로 변경 → 관리자 승인 페이지에 거부 사유 표시

6. 장애 리포트 페이지
   ↓ 발생 일시, 원인, 조치 내역, 현재 상태 정리된 리포트 표시
   ↓ 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 비용 이상 감지 목록 조회 | Notion API에서 Fake 데이터를 가져와 평소 대비 비용 급증 항목을 카드 리스트로 표시 | 시스템의 시작점, 사용자가 이상 여부를 파악하는 핵심 기능 | 이상 감지 목록 페이지 |
| **F002** | AI 분석 결과 표시 | 선택한 이상 항목의 비용 급증 구간, 원인, AI 분석 내용을 상세하게 표시 | 관리자가 조치를 결정하기 위한 정보 제공 핵심 기능 | AI 분석 상세 페이지 |
| **F003** | AI 조치 제안 표시 | AI가 제안한 구체적인 조치 목록을 표시하고 관리자 승인 요청 트리거 | 자동화 조치 실행 전 관리자 검토 필요 | AI 분석 상세 페이지 |
| **F004** | 관리자 승인/거부 처리 | 승인 대기 중인 AI 조치 제안을 승인하거나 거부하는 기능 | 자동 조치 실행의 관문으로 보안 및 책임 소재 명확화 | 관리자 승인 페이지 |
| **F005** | 자동 조치 실행 (Fake) | 관리자 승인 후 AI 제안 조치를 자동으로 실행하는 시뮬레이션 | 시스템 자동화 핵심 가치 검증 | 관리자 승인 페이지 |
| **F006** | 장애 리포트 생성 및 조회 | 조치 완료 후 발생 일시, 원인, 조치 내역, 현재 상태를 정리한 리포트 자동 생성 및 표시 | 사후 분석 및 감사 추적을 위한 필수 기록 | 장애 리포트 페이지 |

### 이상 감지 목록 페이지

> **구현 기능:** `F001`, `F011` | **메뉴 위치:** 헤더 최상단 "이상 감지 목록"

| 항목 | 내용 |
|------|------|
| **역할** | 시스템 랜딩 페이지로, AWS 데이터 전송비용 이상 항목을 한눈에 파악하는 핵심 대시보드 |
| **진입 경로** | 로그인 성공 후 자동 이동, 또는 헤더 메뉴 "이상 감지 목록" 클릭 |
| **사용자 행동** | 비용 급증 항목 카드 목록을 확인하고, 상세 분석이 필요한 항목 카드를 클릭 |
| **주요 기능** | - Notion API에서 Fake 이상 감지 데이터 로드 및 표시<br>- 항목별 카드 표시 (서비스명, 급증 비율, 감지 일시, 현재 상태 배지)<br>- 상태별 색상 구분 (감지됨: 빨강, 승인대기: 주황, 조치완료: 초록, 거부됨: 회색)<br>- 항목 카드 클릭 시 AI 분석 상세 페이지로 이동<br>- **상세 분석 보기** 버튼 (카드 내) |
| **다음 이동** | 카드 클릭 → AI 분석 상세 페이지 |

### AI 분석 상세 페이지

> **구현 기능:** `F002`, `F003`, `F011` | **메뉴 위치:** 이상 감지 목록 페이지에서 카드 클릭으로 진입

| 항목 | 내용 |
|------|------|
| **역할** | 선택한 비용 급증 이상 항목의 원인을 AI가 분석한 결과와 조치 제안을 표시하고 관리자 승인 요청을 트리거하는 페이지 |
| **진입 경로** | 이상 감지 목록 페이지에서 카드 클릭 |
| **사용자 행동** | AI 분석 내용과 비용 급증 구간 정보를 검토한 후, AI 제안 조치를 관리자 승인에 올림 |
| **주요 기능** | - 이상 항목 기본 정보 표시 (서비스명, 감지 일시, 비용 급증 수치)<br>- 비용 급증 발생 구간 시각화 (From → To 구간, 비용 증가량)<br>- AI 분석 결과 표시 (급증 원인 텍스트, 관련 AWS 리소스)<br>- AI 제안 조치 목록 표시 (조치 항목, 예상 효과)<br>- 현재 상태 표시 (승인대기 상태면 버튼 비활성화)<br>- **관리자 승인 요청** 버튼 (상태가 "감지됨"일 때만 활성화) |
| **다음 이동** | 승인 요청 클릭 → 상태 "승인대기"로 변경 후 동일 페이지 유지, 목록으로 돌아가기 → 이상 감지 목록 페이지 |

### 관리자 승인 페이지

> **구현 기능:** `F004`, `F005`, `F011` | **메뉴 위치:** 헤더 메뉴 "관리자 승인"

| 항목 | 내용 |
|------|------|
| **역할** | 관리자가 AI 제안 조치를 검토하고 승인 또는 거부하는 전용 페이지. 승인 시 자동 조치(Fake)가 실행됨 |
| **진입 경로** | 헤더 메뉴 "관리자 승인" 클릭 |
| **사용자 행동** | 승인 대기 중인 조치 항목 목록을 검토하고 각 항목에 대해 승인 또는 거부 처리 |
| **주요 기능** | - 승인 대기 중인 조치 항목 목록 표시 (이상 항목명, AI 제안 조치 요약, 요청 일시)<br>- 각 항목 클릭 시 AI 분석 내용 및 제안 조치 상세 확인 패널 표시<br>- **승인** 버튼 클릭 시 자동 조치 실행 시뮬레이션 후 장애 리포트 자동 생성<br>- **거부** 버튼 클릭 시 거부 사유 입력 모달 표시 후 상태 "거부됨"으로 변경<br>- 처리 완료 항목은 목록 하단에 상태 배지와 함께 표시 |
| **다음 이동** | 승인 처리 완료 → 장애 리포트 페이지로 자동 이동, 거부 처리 → 동일 페이지 유지 |

---

### 장애 리포트 페이지

> **구현 기능:** `F006` | **메뉴 위치:** 헤더 메뉴 "장애 리포트"

| 항목 | 내용 |
|------|------|
| **역할** | 조치가 완료된 이상 항목에 대한 장애 리포트를 표시하는 페이지. 사후 분석 및 감사 추적용 |
| **진입 경로** | 관리자 승인 페이지에서 승인 처리 완료 후 자동 이동, 또는 헤더 메뉴 "장애 리포트" 클릭 |
| **사용자 행동** | 장애 리포트 목록을 확인하고 특정 리포트를 클릭하여 상세 내용을 검토 |
| **주요 기능** | - 장애 리포트 목록 표시 (리포트 제목, 발생 일시, 현재 상태)<br>- 리포트 상세 내용: 문제 발생 일시 및 시간, 데이터 비용 급증 원인 요약, 취해진 조치 내역 목록, 조치 후 현재 상태<br>- 리포트별 전체 워크플로우 타임라인 표시 (감지 → 분석 → 승인 → 조치완료)<br>- **목록으로 돌아가기** 버튼 |
| **다음 이동** | 목록으로 돌아가기 → 장애 리포트 목록, 이상 감지 목록 이동 → 이상 감지 목록 페이지 |

## 데이터 모델

### AnomalyAlert (이상 감지 항목)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| serviceName | AWS 서비스명 (예: EC2, S3, CloudFront) | String |
| detectedAt | 이상 감지 일시 | DateTime |
| normalCost | 평소 데이터 전송 비용 | Number |
| anomalyCost | 급증한 데이터 전송 비용 | Number |
| increaseRate | 비용 증가율 (%) | Number |
| fromRegion | 비용 급증 발생 출발 구간 | String |
| toRegion | 비용 급증 발생 도착 구간 | String |
| status | 현재 상태 (detected/analyzing/pending/resolved/rejected) | Enum |
| notionPageId | Notion 원본 데이터 페이지 ID | String |

### AiAnalysis (AI 분석 결과)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| alertId | 연결된 이상 감지 항목 | → AnomalyAlert.id |
| rootCause | AI 분석 원인 텍스트 | String |
| affectedResources | 영향받은 AWS 리소스 목록 | String[] |
| suggestedActions | AI 제안 조치 목록 | JSON |
| analyzedAt | 분석 완료 일시 | DateTime |

### ApprovalRequest (관리자 승인 요청)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| alertId | 연결된 이상 감지 항목 | → AnomalyAlert.id |
| requestedAt | 승인 요청 일시 | DateTime |
| decidedAt | 승인/거부 처리 일시 | DateTime |
| decision | 결정 (approved/rejected) | Enum |
| rejectionReason | 거부 사유 (거부 시) | String |

### IncidentReport (장애 리포트)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| alertId | 연결된 이상 감지 항목 | → AnomalyAlert.id |
| title | 리포트 제목 | String |
| occurredAt | 문제 발생 일시 | DateTime |
| resolvedAt | 조치 완료 일시 | DateTime |
| rootCauseSummary | 원인 요약 | String |
| actionsTaken | 취해진 조치 내역 | String[] |
| currentStatus | 현재 상태 설명 | String |

### User (사용자)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| name | 사용자 이름 | String |
| email | 이메일 (로그인 ID) | String |
| passwordHash | 암호화된 비밀번호 | String |
| createdAt | 가입 일시 | DateTime |
